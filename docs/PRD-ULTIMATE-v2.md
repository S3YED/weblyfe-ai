## 🏗️ PRODUCT ARCHITECTURE

### The Appie System (4-Appie Fleet)
```
CEO / Orchestrator (Appie-1)
    │
    ├── CMO / Herald (Appie-2) — Marketing & Content
    │       → Content Factory, SEO, Social
    │
    ├── CTO / DevOps (Appie-3) — Technical & Systems
    │       → GitHub, CI/CD, Infrastructure
    │
    └── CFO / Business Intelligence (Appie-4) — Finance & Ops
            → Stripe, Airtable, Reporting
```

### How Each Appie Works
```
User Input (Telegram)
       ↓
  OpenClaw Gateway
       ↓
  Appie (AI Brain) ← MiniMax M2.7
       ↓
  Skills (55+ pre-built)
       ↓
  Tools (gog, browser, code, memory)
       ↓
  Brain System (Context + Memory)
```

### Self-Improvement Loop
```
1. MEMORY: Appie stores everything it learns
2. SKILLS: Writes new skills from lessons
3. EXECUTE: Does the work
4. REVIEW: Scores own performance (1-10)
5. IMPROVE: Updates brain based on results
6. REPEAT: Gets better every iteration
```

### Knowledge Architecture
- **Brain:** Semantic memory layer (Pinecone vector DB)
- **Graphify:** Codebase structure graph (Python AST-based)
- **Context Engineering:** JIT loading of relevant context only
- **Daily Memory:** Automatic consolidation of learnings

## 🧠 APPENDIX: KEY PRODUCT DECISIONS

### Why Telegram First?
- 1B+ users, zero friction to start
- Already in everyone's pocket
- Voice notes, text, files all supported
- No new app to download or learn
- Notification-native for async work

### Why Self-Hosted?
- Data privacy (customer data stays with client)
- No subscription to tool makers
- Full control over the AI's behavior
- Perpetual license, not rent

### Why MiniMax M2.7?
- Fast inference (~50 tokens/sec)
- $0.60/month for 60M tokens (vs $50+ for GPT-4)
- Good reasoning at low cost
- Available via gog CLI natively

### Why OpenClaw over LangChain/Autogen?
- Built for non-technical users
- Telegram-first architecture
- Pre-built skills marketplace
- Fleet management out of the box
